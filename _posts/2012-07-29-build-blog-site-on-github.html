在 github 上搭建 blog

一直想自己也能持续的写 blog，总觉得无啥可写。现在回想几年前的自己，一片空白，是该养成一个记录习惯了。并且还能通过 blog 进行分享，总是好的。

之前有了解到可在 github 上搭建 blog，并且写 blog 的方式正好是我喜欢的：

  • 通地简单直观的标记语法生成静态html
  • 免费
  • 定制性高

其实就第一条理由，另外就是对 github 有好感。所以就在上面搞了，顺便记录一下过程。

过程

  1. 在 github 有帐号，没有就注册一个
  2. 创建名为 username.github.com 的 repository，名字是这样时，github 才会将 master 分支的内容做为你的 github pages. 对于项目的页面，是将 gh-pages 分支的内容做为页面的。
  3. 在master分支上写入内容并上传，(假设就一个 index.html文件) 
    mkdir xxx.github.com && cd xxx.github.com
echo "XXX's home page" > index.html
git init
git add index.html
git commit -m "first commit"
git remote add origin git@github.com:XXX/XXX.github.com.git
git push

    然后过10分钟，就可以在 XXX.github.com上看到你刚写的那个 index.html 了。

其实这样就已经可以用了，我也想过直接将 org publish 成 html，然后再加 shell 脚本成生个主页和 RSS 就行了 (这样想是以为 jekyll 只支持 textile 和 markdown, 其实直接写 html 是理所当然的支持的)。但有现成的 jekyll，直接用就行了。 jekyll 的模板可参考 https://github.com/mojombo/jekyll/wiki/sites , 我自己用的是： https://github.com/flyerhzm/flyerhzm.github.com.git 这个，再稍加修改的，尽量简单。

遇到的问题：

  • 发现 jekyll 不支持 emacs org，想通过 pandoc 将 org 转成 markdown，后来发现 pandoc 的输入只能是: native, json, markdown, textile, rst, html, latex，只有在输出里才支持 emacs org.
  • 在 jekyll 中要使用 paginator 必需在 config.yml 中加入 paginate: true，我是从别人那里看到的写的 paginate: 5。(反正能用，没仔细了解过文档)
  • 在将其发布到 github 中时，第一次将页面发在了 gh-pages 分支中。 之后发现文档应该放到 master 下，所以放到 master 下并推到 github, 然后将远程的 gh-pages 分支删除 ~git push origin :gh-pages~。 然后 zjuliubin.github.com 就 404 了，等了很久(>20m)都没出来。不知道算不算是 bug，然后删除 repository 搞定。
